v1.0 — 22 JUNE 2026
================================================================================

SUBMISSION SYSTEM
-------------------------------------
- Full 8-state designer → CS → admin workflow for both C&CM and Standard briefs
  State order: In Queue → In Progress → Submitted → Awaiting Review →
  Revision Requested → Re-submitted → CS Approved → Approved
- Designers can submit deliverables individually or in bulk via a picker
- CS can approve or request revision per submission, with a message
- Revision counter tracks how many rounds a submission has gone through
- All state transitions logged to Activity Log and trigger notifications
- Standard brief: submission is project-level (one approval covers all deliverables)
- POSM: submission is per-channel per-customer; project cascades to Approved
  when all channels are approved

POSM CHANNEL SYSTEM
-------------------------------------
- UAE channel tracks approval per customer
- Gulf channel (KSA, Kuwait, Bahrain, Oman, Qatar) tracks approval per country
- Each channel has its own submission → revision → CS approval flow
- Project cannot reach Approved status until every active channel is approved
- POSM revision count tracked separately from C&CM revision count
- "Begin POSM" button appears after concept/KV phase is completed

PROJECT APPROVAL SYSTEM
-------------------------------------
- Admin can mark a project as Approved (final lock) from the project detail page
- Approval requires confirmation modal — cannot be undone
- Approved projects are locked: CS cannot change deliverable statuses or remove
  reference files; all mutating routes return 403
- Approved projects move to a dedicated "Approved Projects" tab on all three
  dashboards (CS, Team Lead, Management) and are removed from active lists
- Approved date and approving admin are stamped on the project record

CONCEPT & KEY VISUAL — MERGED
-------------------------------------
- Concept and KV are now always submitted as a single unit
- Create form: one combined "Concept & KV" toggle replaces two separate toggles;
  single requirements field, single deadline, single options count
- Project detail page: one merged "Concept & KV" block with both designer
  assignment fields (Concept Designer and KV Designer remain separate)
- Revision picker shows a single "Concept & KV" row; selecting it revises both
- Progress bar logic updated: region selection no longer required for 100%

STANDARD BRIEF — EDIT FIX
-------------------------------------
- Editing a standard brief no longer wipes deliverable progress
- Deliverables are now upserted by name: existing status/revision data is
  preserved; only deadline and team tags are updated
- Previously saved deliverables now correctly appear when reopening the edit form
  (fixed a silent JSON.parse failure caused by quote escaping in HTML attributes)

NOTIFICATIONS — BULK OPERATIONS
-------------------------------------
- New "Archive All" button in the inbox: archives every notification at once
  and immediately moves them to the Archived tab without a page refresh
- New "Delete All" button in the Archived tab: permanently removes all archived
  notifications in one click
- Archived tab updates in real time after Archive All — no refresh needed
- If Archived tab was previously empty, its toolbar is injected automatically

BUG FIXES
-------------------------------------
- Approval confirmation modal: title was rendering in all-caps due to global
  CSS rule — fixed with explicit text-transform override
- Approval modal background was transparent (--surface CSS variable undefined) —
  fixed to use --white
- Approval confirm button was unresponsive — callback reference was being nullified
  before it could be called; fixed by saving reference before closing modal
- Deliverable status dropdowns are now read-only badges for CS on approved projects
- Reference file Remove button is hidden for CS on approved projects
